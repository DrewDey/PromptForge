Three root causes diagnosed and resolved: **(a)** missing readiness probe — without it, pods entered the GCE LB backend pool immediately on container start rather than after Gunicorn had initialized, causing a 5–15 second window of dropped requests on every rollout; **(b)** no graceful shutdown handling — GCE LB takes 30–60 seconds to drain a backend, but pods were being SIGTERM'd with no preStop delay, causing requests to arrive at a pod that had already closed its listener throughout the drain window; **(c)** migration-before-code sequencing — running Alembic before deploying new pod code meant old pods were running against the new schema for the entire rollout duration, creating a forward-incompatibility window for any migration that changes column nullability or names. Four passes: **(1)** failure mode diagnosis at the LB and pod lifecycle level with the specific mechanism for each of the three errors; **(2)** Kubernetes Deployment manifest update with readiness probe (httpGet /health, initialDelaySeconds: 5, failureThreshold: 3), liveness probe (initialDelaySeconds: 30, conservative restart threshold), preStop lifecycle hook (sleep 15), terminationGracePeriodSeconds: 75 (LB drain window + preStop + buffer), rolling update strategy maxSurge: 1 / maxUnavailable: 0, PodDisruptionBudget minAvailable: 2 with reasoning for each value; **(3)** expand/contract migration pattern with two complete Alembic examples — adding a NOT NULL column (Phase 1: add nullable + write in new code, Phase 2: backfill + constraint after pods cycled) and renaming a column (four phases: add new column + populate, dual-write in code, add NOT NULL after old pods gone, drop old column) — including CREATE INDEX CONCURRENTLY detail for 2.4M-row production tables and the Alembic transaction context caveat; **(4)** Kubernetes-native canary deployment using label-selector traffic split (3 stable replicas + 1 canary = ~25% traffic, no service mesh required), kubectl scale to zero for instant abort, automated rollback watchdog (Python CronJob checking Datadog error rate against 2% threshold, scales canary to zero and posts to Slack).
