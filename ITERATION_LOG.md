**Why this approach (design rationale).** Backlog Polish #1 named three candidate prose labels — "ask" / "approach" / "brief" — and left the pick to the iteration. Chose **ask / response** over ask / approach / brief for a specific reason: the detail page is read as a conversation between an author and a model ("they said X, the model answered Y, then they said Z"). Conversational vocabulary (ask ↔ response) matches that mental model. "Approach" frames the entry as the author's strategy rather than their input; "brief" frames it as a formal spec deliverable. Neither feels right when the step payload is a casual, inline "write me three versions and explain the tradeoffs" question. Response pairs naturally with ask; result does not (result is a verdict, which is what Final Output at the top of the page now represents). On where the label-pick logic lives: considered three alternatives. (a) Keep the `label` prop on `StepContent` and have callsites pass the right string — rejected because that leaves four places in the file where a future editor could accidentally pass "prompt" as the prose label and undo the work; centralizing the pick means the vocabulary is enforced by type system + dispatcher, not by editor discipline. (b) Push the vocabulary split down into `Prose` itself (add an implicit default of "ask"/"response" based on variant) — rejected because `Prose` is a generic editorial card component (it's rendered once per detail page now, but could be reused in the Build page builder or in a future comments surface) and baking step-flow vocabulary into it makes that reuse awkward. The component stays vocabulary-neutral; the dispatcher is the vocabulary gatekeeper. (c) Keep labels on callsites but use a shared constants object (`CHROME_LABELS.prose.prompt = 'ask'`) — rejected as ceremony over substance; the dispatcher is already the single-source-of-truth surface this iteration needs, and a constants object would just be a second indirection. CHOSE the dispatcher-owned approach: `StepContent` takes `variant` (semantic role) and `text` and does the full lookup internally. This also means future classifier tweaks automatically route to the right label — if a natural-language payload flips to `code` after a heuristic tune, its eyebrow becomes "prompt" without anyone touching callsites. On the inline copy choices: "Expand to reveal the ask behind it" was chosen over "Expand to reveal what they asked" because "the ask" is a noun phrase that matches the eyebrow vocabulary pattern exactly — the subtitle is introducing the reader to the same word they'll see on the card. "Show the ask behind this step" over "Show the ask that produced this step": "behind" suggests the ask is the thing this step is built on (accurate — it IS the input payload), whereas "produced" implies causation the reader hasn't earned yet when the disclosure is still collapsed. For the single-prompt h2 "The Ask" (vs "The Approach"): the field below the h2 renders the author's literal natural-language input — calling that "The Approach" would suggest the author is explaining their methodology, but on the no-chain branch the value is usually a single direct question to the model. "The Ask" is honest about what follows.
